Odladění návrhu

Když už máme naše tabulky a relace hotové, je načase naši databázi vyzkoušet na testovacích datech. Databázi naplníme testovacími daty a s těmito daty budeme pracovat (vytvářet dotazy, přidávat záznamy, měnit existující záznamy, mazat záznamy...). Tímto způsobem můžeme objevit poteciální problémy, např. že nám chýbějí nějaké atributy, nebo že máme přebytečné atributy (dají se zjistit z jiné tabulky) nebo že databáze obsahuje tabulku, ktetá se musí rozdělit na dvě. Níže vám ukáži, jak databázi daty naplnit a jak s nimi pracovat , ale nejdříve si musíme něco říci o jazyku SQL.

SQL

SQL je zkratka (Structured Query Language) česky strukturovaný dotazovací jazyk. Pomocí tohoto jazyku můžeme získávat, vkládat, měnit nebo mazat data z databáze. Také nám dovoluje vytvářet objekty (tabulky) a nastavovat přístupová práva.

Několik zaklaních SQL příkazů

Příkazy pro manipulaci z daty (DML - Data Manipulation Language)

Tyto příkazy slouží abychom mohly s databází pracovat.

  • SELECT - umožňuje výběr dat z databáze
  • INSERT - vkláda data do databáze
  • UPDATE - mění existující data v databázi
  • DELETE - odstraňuje data z databáze

Příkazy pro definici dat (DDL - Data Definition Language)

Těmito příkazy se vytvářejí objekty (tabulky) v databázi.

  • CREATE - vytváří nové objekty v databázi
  • ALTER - mění již existující objekty
  • DROP - odstraní objekty v databázi

Příkazy pro řízení dat (DCL - Data Control Language) a příkazy pro ovládání transakcí (TCL - Transaction Control Language)

Tyto příkazy slouží k nastavení přístupových práv a řízení transakcí.

  • GRANT - přidělí oprávění uživateli k určitým objektům
  • REVOKE - odebírá práva uživateli
  • BEGIN - spustí transakci
  • COMMIT - odsouhlasení transakce
  • ROLLBACK - zruší transakci a vrátí se do stavu před ní.
