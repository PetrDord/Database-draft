Odladění návrhu

Když už máme naše tabulky a relace hotové, je načase naší databázi vyzkoušet na testovacích datech. Databázi naplníme testovacími daty a s těmito daty budeme pracovat (vytvářet dotazy, přidávat záznamy, měnit existující zaznamy, mazat záznamy...). Tímto způsobem můžeme objevit poteciální problémy, např. že nám chýbějí nějaké atributy, nebo že máme přebytečné atributy (dají se zjistit z jiné tabulky) nebo že databáze obsahuje tabulku, ktetá se musí rozdělit na dvě. Níže vám ukáži, jak databázi daty naplnit a jak s nimi pracovat , ale nejdříve si musíme něco říci o jazyku SQL.

SQL

SQL je zkratka (Structured Query Language) česky strukturovaný dotazovací jazyk. Pomocí tohoto jazyku můžeme získávat, vkládat, měnit nebo mazat data z databáze. Také nám dovoluje vytvářet objekty (tabulky) a nastavovat přistupová práva.

Několik zaklaních SQL příkazů

Příkazy pro manipulaci z daty (DML - Data Manipulation Language)

Tyto příkazy slouží k získaní dat z databáze a pro jejich úpravu.

  • SELECT - vybírá data z databáze, umožňuju výběr pod množin a řazení dat
  • INSERT - vkláda data do databáze
  • UPDATE - mění data v databázi
  • DELETE - odstraňuje záznamy z databáze

Příkazy pro definici dat (DDL - Data Definition Language)

Těmito příkazy se vytvářejí struktury databáze - tabulky, indexy, pohledy a další objekty.

  • CREATE - vytoří nový objekt
  • ALTER - mění již existující objekty
  • DROP - odstraní objekt

Příkazy pro řízení dat (DCL - Data Control Language)

Tyto příkazy slouží k nastavení přístupových práv a řízení transakcí.

  • GRANT - přidělí oprávění uživateli k určitým objektům
  • REVOKE - odnětí práv uživateli
  • BEGIN - zahájení transakce
  • COMMIT - potvrzení transakce
  • ROLLBACK - zrušení transakce, návrat do původního stavu
